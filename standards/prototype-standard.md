# Prototype Standard

## Purpose

Prototypes should validate actual JobPilot workflows, not merely decorate a page.

## Required For Core Features

- Related `US-xxx`.
- Entry points.
- Page and state flow.
- Key screens.
- Default, loading, empty, error, success, disabled, and manual handoff states.
- High-fidelity acceptance checklist.

## UX Benchmark

Before high-fidelity work:

- Review relevant products or patterns.
- Store notes in `research/ux/`.
- Explain what JobPilot should borrow and avoid.

## Visual Quality Bar

- Interface should feel like a credible commercial product.
- Avoid generic AI demo styling.
- Prefer clear hierarchy, restrained color, readable density, and complete states.
- Do not hide important product decisions behind visual polish.
