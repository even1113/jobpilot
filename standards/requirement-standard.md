# Requirement Standard

## US ID

- Use `US-001`, `US-002`, `US-003`.
- Use `US-001-A` for explicit subtasks only.
- Never reuse deprecated IDs.
- Functional commits and PRs must include the related US ID.

## Required Sections

Each formal requirement must include:

- Status.
- Target user.
- Scenario.
- User story.
- Scope and non-goals.
- Acceptance criteria.
- UX/prototype links.
- Technical notes.
- Test notes.
- Dependencies.
- Risks.

## Ready For UX

A requirement is ready for UX when:

- Target user and scenario are clear.
- Core workflow is described.
- Main success and failure states are known.
- Open questions are listed.

## Ready For Development

A requirement is ready for development when:

- Acceptance criteria are testable.
- Prototype or UI notes exist when UI is involved.
- Dependencies and risks are explicit.
- `quality/definition-of-ready.md` is satisfied.
