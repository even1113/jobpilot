# Prototype

`prototype/` stores runnable static prototypes and feature-level prototype specs.

## Current Runnable Prototypes

- `index.html`: combined C-side, Web, and plugin prototype.
- `miniprogram.html`: mini-program style prototype.
- `sidebar.html`: browser extension sidebar prototype.
- `原型.md`: original prototype prompt/notes.

## Feature Prototype Specs

Use `features/feature-prototype-template.md` for each `US-xxx` flow that needs UX or high-fidelity validation.

## Rules

- Keep existing HTML demos runnable.
- Do not replace a UX decision with undocumented visual changes.
- Each core feature prototype must document entry points, page transitions, core states, empty states, loading states, error states, and acceptance points.
- Link feature prototype specs from `requirements/feature-map.md`.
