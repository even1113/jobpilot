# Definition of Ready

A feature is ready for development only when:

- [ ] It has a stable `US-xxx` ID in `requirements/backlog.md`.
- [ ] The requirement has target user, scenario, scope, and non-goals.
- [ ] Acceptance criteria are testable.
- [ ] UI work has a prototype spec or explicit reason why it does not need one.
- [ ] Dependencies and risks are known.
- [ ] Safety, privacy, and platform automation constraints are listed when relevant.
- [ ] The expected test approach is clear.

If any item is missing, keep the requirement in `draft` or `ready-for-ux`.
