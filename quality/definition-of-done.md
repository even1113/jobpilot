# Definition of Done

Work is done only when:

- [ ] It satisfies the related `US-xxx` acceptance criteria.
- [ ] Implementation matches the requirement and prototype, or deviations are documented.
- [ ] Important success, empty, loading, error, and manual handoff states are handled.
- [ ] Relevant tests or checks have run.
- [ ] Documentation, indexes, and feature map are updated when needed.
- [ ] Git branch and commit/PR text follow `standards/git-standard.md` if Git actions are performed.
- [ ] Remaining risks are clearly reported.
