# JobPilot Docs Index

`docs/` stores product-level and technical-level documents. Feature-level work should live in `requirements/` and `prototype/features/` so large documents do not become the only source of truth.

## Current Documents

- `PRD_JobPilot_v1.0.md`: product requirements and MVP scope.
- `TDD_JobPilot_v1.0.md`: technical design and architecture draft.

## Subdirectories

- `product/`: product strategy, market notes, roadmap notes, and PRD supplements.
- `technical/`: architecture notes, module design, API design, and data model supplements.
- `decisions/`: decision records for important product or technical choices.

## Rules

- Keep PRD/TDD as high-level references.
- Put every formal feature in `requirements/` with a `US-xxx` ID.
- Put implementation-changing architecture decisions in `docs/decisions/`.
- Update this index when adding important documents.
