## 二、产品概述

### 2.1 产品定位

**一句话描述**：JobPilot 是一款轻量化的求职全流程管理工具，帮助求职者聚合岗位、追踪投递、准备面试，让求职过程清晰可控。

### 2.2 目标用户

| 用户类型 | 特征 | 核心痛点 |
|----------|------|----------|
| 应届毕业生 | 首次求职，缺乏经验，投递量大 | 不知道投了哪些、进度混乱、简历不会写 |
| 职场人 | 有工作经验，寻求跳槽，投递精准 | 面试时间难管理、多平台信息分散、缺乏复盘 |

### 2.3 核心价值主张

1. **聚合**：多平台岗位统一搜索，告别来回切换
2. **追踪**：投递进度看板化，一目了然
3. **提醒**：面试时间自动提醒，不再错过
4. **智能**：AI 简历诊断 + JD 匹配评分，提升求职效率

### 2.4 产品形态

| 端 | 定位 | 核心场景 |
|----|------|----------|
| 微信小程序 | **主入口** | 随时查看投递进度、面试提醒、快速搜索岗位 |
| Web 端 | **辅助入口** | 简历编辑、数据看板、批量操作等重操作场景 |

两端数据实时同步，账号互通。

#### 3.2.1 F1 - 用户登录注册

| 项目 | 说明 |
|------|------|
| 功能描述 | 用户通过微信授权登录，无需额外注册 |
| 优先级 | P0 |
| 端 | 小程序 + Web |

**需求细节：**

- 小程序端：用户点击"微信登录"按钮，授权后自动完成登录
- Web 端：展示小程序码，用户扫码后自动登录
- 首次登录自动创建用户记录，昵称和头像从微信获取
- 登录态失效后自动跳转登录页，无需手动重新登录
- 同一微信账号在小程序和 Web 端登录后，数据完全互通

**验收标准：**

- [ ] 小程序一键登录成功，返回用户信息
- [ ] Web 端扫码登录成功，与小程序账号互通
- [ ] 登录态失效后自动跳转登录页

#### 3.2.2 F2 - 岗位聚合搜索

| 项目 | 说明 |
|------|------|
| 功能描述 | 聚合 Boss 直聘、拉勾等平台岗位，统一搜索浏览 |
| 优先级 | P0 |
| 端 | 小程序 + Web |

**需求细节：**

- 搜索条件：关键词（必填）、城市（可选）、薪资范围（可选）、来源平台（可选）
- 搜索结果列表展示：岗位名称、公司名称、薪资范围、城市、来源平台标签、发布时间
- 支持分页加载，每页 20 条
- 点击岗位卡片可查看详情：完整 JD、任职要求、公司简介
- 详情页提供"加入看板"按钮，一键创建投递记录
- 岗位数据来源于平台数据采集，前端不做实时爬取
- MVP 阶段仅接入 Boss 直聘和拉勾两个平台

**验收标准：**

- [ ] 输入关键词可搜索到岗位结果
- [ ] 搜索结果包含来自 Boss 直聘和拉勾的岗位
- [ ] 可按城市、薪资范围筛选
- [ ] 岗位详情页展示完整信息
- [ ] "加入看板"可成功创建投递记录

#### 3.2.3 F3 - 投递进度看板

| 项目 | 说明 |
|------|------|
| 功能描述 | 以看板视图管理投递进度，支持状态流转 |
| 优先级 | P0 |
| 端 | 小程序 + Web |

**需求细节：**

**状态定义：**

| 状态 | 含义 | 颜色 |
|------|------|------|
| 待投递 | 已收藏但未投递 | 灰色 |
| 已投递 | 已提交简历 | 蓝色 |
| 面试中 | 进入面试环节 | 橙色 |
| 已 Offer | 收到录用通知 | 绿色 |
| 已结束 | 被拒绝/已放弃 | 红色 |

**状态流转规则：**

```
待投递 → 已投递 → 面试中 → 已Offer
                    ↓           ↓
                 已结束      已结束
```

- 允许回退到上一状态（如"面试中"→"已投递"）
- "已结束"为终态，不可回退
- 每次状态变更记录时间戳

**看板视图：**

- 横向展示 5 个状态列，每列显示对应状态的投递卡片
- 卡片信息：岗位名称、公司名称、薪资、停留时长
- Web 端支持拖拽卡片切换状态
- 小程序端通过卡片操作按钮切换状态（长按或右滑菜单）

**列表视图：**

- 按更新时间倒序排列
- 支持按状态筛选
- 支持关键词搜索

**操作：**

- 新增投递记录：从岗位详情页"加入看板"或手动添加
- 编辑投递记录：修改备注、面试时间
- 删除投递记录：二次确认后删除
- 批量操作：批量修改状态、批量删除（Web 端）

**验收标准：**

- [ ] 看板视图正确展示 5 个状态列及对应卡片
- [ ] 状态流转符合规则，非法流转被拦截
- [ ] Web 端拖拽切换状态正常
- [ ] 小程序端按钮切换状态正常
- [ ] 新增/编辑/删除投递记录正常
- [ ] 状态变更记录时间戳

#### 3.2.4 F4 - 面试提醒

| 项目 | 说明 |
|------|------|
| 功能描述 | 设置面试时间，到时自动提醒 |
| 优先级 | P0 |
| 端 | 小程序 + Web |

**需求细节：**

- 在投递记录中可设置面试时间（日期+时间）
- 面试时间设置后，投递记录自动流转到"面试中"状态
- 提醒方式：
  - 小程序：微信订阅消息推送（面试前 1 天 + 面试前 2 小时）
  - Web：浏览器通知（面试前 2 小时）
- 提醒内容：公司名称、岗位名称、面试时间
- 面试看板：单独展示所有即将到来的面试，按时间排序

**验收标准：**

- [ ] 设置面试时间后状态自动变为"面试中"
- [ ] 小程序订阅消息按时推送
- [ ] Web 浏览器通知按时推送
- [ ] 面试看板按时间排序展示

#### 3.2.5 F5 - AI 简历诊断

| 项目 | 说明 |
|------|------|
| 功能描述 | AI 多维度评估简历质量，给出具体修改建议 |
| 优先级 | P1 |
| 端 | Web 为主，小程序查看结果 |

**需求细节：**

- 用户上传简历（PDF/Word）或粘贴文本
- AI 从以下维度评分（每项 0-100 分）：
  - **内容完整度**：是否涵盖教育、经历、技能等核心模块
  - **关键词匹配**：是否包含目标岗位常见关键词
  - **成果量化**：是否有可量化的成果描述（如"提升30%"）
  - **格式规范**：结构是否清晰、排版是否专业
  - **差异化**：是否有独特亮点区分其他候选人
- 综合评分 + 各维度评分 + 具体修改建议
- 支持指定目标岗位方向，AI 按该方向评估关键词匹配
- 诊断结果可保存，支持历史对比

**验收标准：**

- [ ] 上传简历后可获取 AI 诊断结果
- [ ] 诊断结果包含 5 个维度评分 + 综合评分
- [ ] 修改建议具体可操作
- [ ] 可指定目标岗位方向

---

#### 3.2.6 F6 - AI JD 匹配评分

| 项目 | 说明 |
|------|------|
| 功能描述 | AI 评估简历与目标 JD 的匹配程度 |
| 优先级 | P1 |
| 端 | Web 为主，小程序查看结果 |

**需求细节：**

- 用户选择一份简历 + 一个岗位（从看板中选择或粘贴 JD）
- AI 输出：
  - **匹配度评分**：0-100 分
  - **匹配项**：简历中符合 JD 要求的要点
  - **缺失项**：JD 要求但简历中未体现的要点
  - **优化建议**：如何调整简历提高匹配度
- 匹配结果保存在投递记录中，方便对比

**验收标准：**

- [ ] 选择简历和岗位后可获取匹配评分
- [ ] 匹配结果包含匹配项、缺失项、优化建议
- [ ] 匹配结果保存在投递记录中

## 四、非功能需求

| 指标 | 目标值 |
|------|--------|
| 页面/接口平均响应时间 | < 500ms |
| 搜索结果返回时间 | < 1s |

| 并发用户数（MVP） | 500 |
| 小程序首屏加载 | < 2s |

### 4.2 安全要求

| 项目 | 要求 |
|------|------|
| 传输安全 | 全站 HTTPS |
| 用户认证 | 基于微信 OAuth |
| 简历数据 | 加密存储，不可明文落盘 |
| 敏感信息 | 脱敏展示（手机号、邮箱） |
| 输入安全 | 防注入、防 XSS |

### 4.3 可用性要求

| 指标 | 目标值 |
|------|--------|
| 服务可用性 | 99.5% |
| 数据备份 | 每日全量备份 |
| 故障恢复 | < 1 小时 |

### 4.4 兼容性要求

| 端 | 要求 |
|----|------|
| 小程序 | 微信 7.0+ |
| Web | Chrome 80+、Safari 14+、Edge 80+ |
| 移动端适配 | Web 端响应式，适配 375px-428px 宽度 |

## 五、页面与交互设计要点

### 5.1 页面清单

| 页面 | 端 | 说明 |
|------|-----|------|
| 登录页 | 小程序+Web | 微信授权/扫码登录 |
| 看板首页 | 小程序+Web | 投递进度看板，核心页面 |
| 岗位搜索页 | 小程序+Web | 搜索+结果列表 |
| 岗位详情页 | 小程序+Web | JD 详情+加入看板 |
| 投递记录详情 | 小程序+Web | 编辑状态、备注、面试时间 |
| 面试日历 | 小程序+Web | 面试安排时间线 |
| 简历管理 | Web | 简历列表+上传 |
| AI 诊断结果 | Web | 诊断评分+建议 |
| 个人中心 | 小程序+Web | 设置、关于 |

### 5.2 核心交互流程

**流程 1：搜索岗位 → 加入看板**

```
首页点击搜索 → 输入关键词 → 浏览结果列表 → 点击岗位卡片
→ 查看详情 → 点击"加入看板" → 选择初始状态 → 创建成功
→ 跳转看板页查看
```

**流程 2：看板状态流转**

```
看板页 → 拖拽/点击卡片 → 选择目标状态 → 确认
→ 状态更新 → 卡片移动到目标列 → 记录变更日志
```

**流程 3：AI 简历诊断**

```
简历管理页 → 上传简历/粘贴文本 → 选择目标方向
→ 点击"开始诊断" → Loading → 展示诊断结果
→ 查看各维度评分和建议
```

## 七、风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 招聘平台反爬升级 | 岗位数据获取失败 | 高 | 降级为手动添加+浏览器插件导入 |
| 小程序招聘类目审核不通过 | 无法上线 | 中 | 提前确认资质，备选方案为纯 Web |
| AI 诊断质量不稳定 | 用户体验差 | 中 | Prompt 迭代优化，增加评分校准机制 |
| 简历数据泄露 | 法律风险 | 低 | 加密存储、权限控制、安全审计 |
| 用户量增长超出预期 | 服务不可用 | 低 | 容器化部署，快速扩容 |
